10:00 Java 9 Modules

This presentation is by the author of the O'Reilly book "Java 9 modularity". When we are going to use Java 9, there is a big chance that our current apps don't work anymore, because o (a)busing platform internals which are now hidden. Or unresolved platform modules, like the non SE modules.

11:00 The seven more deadly sins of microservices

Anti patterns for micro services. Updated version with lessons learned.

11:00 The seven deadly sins of microservices

  1. Lust - Using the (unevaluated) latest technology
  2. Gluttony - Communication lock-in
  3. Greed - What's mine is mine (within the organization)
  4. Sloth - Getting lazy with NFRS
  5. Wrath - Blowing up when bad things happen
  6. Envy - The shared single domain (and data store) fallacy
  7. Pride - Testing in the world of transience
Will not discuss them all. The good thing is that it is down to earth. For instance the first one, is about evaluating what your goals are for microservices, with the correct architectural model and DevOps process in place. And also evaluating the technology, for instance with the help of the Spine model. Take for instance the decision about containers, to docker or not to docker.
If your application looks like a giant tire fire, and you try to split it up and put it in a container, you just get a fire in a container.
Ook praktische tips en tools. End the presentation with some great books to read.
